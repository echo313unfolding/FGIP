# Proposed Agents

Agent specifications generated by `AgentFactory` when capability gaps are detected.

## Directory Structure

```
agents/
└── proposed/
    ├── AGENT-SPEC-{DATE}-{edge-type}.json      # Machine-readable spec
    └── AGENT-SPEC-{DATE}-{edge-type}_template.md  # Human-readable template
```

## Specification Format

```json
{
  "agent_id": "AGENT-SPEC-{DATE}-{edge-type}",
  "edge_type": "APPOINTED_BY",
  "data_sources": ["congress.gov", "whitehouse.gov"],
  "tier": 0,
  "estimated_complexity": "medium",
  "dependencies": ["fgip.agents.base"],
  "generated_by": "AgentFactory",
  "generated_at": "2026-02-24T12:00:00Z"
}
```

## Workflow

1. `GapDetector` identifies missing edge types in graph
2. `AgentFactory` generates specification for new agent
3. Specs written to `agents/proposed/`
4. Human reviews and approves specification
5. Developer implements agent in `fgip/agents/`
6. Agent registered in `fgip/agents/__init__.py`

## Implementing a Proposed Agent

```bash
# 1. Read the specification
cat agents/proposed/AGENT-SPEC-20260224-appointed-by.json

# 2. Create the agent file
touch fgip/agents/appointed_by.py

# 3. Follow the template
cat agents/proposed/AGENT-SPEC-20260224-appointed-by_template.md

# 4. Register in __init__.py
# 5. Test with dry-run
python3 -m fgip.cli agent run appointed-by --dry-run
```
